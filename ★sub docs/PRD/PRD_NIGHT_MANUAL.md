# 바이럴 기능 요구사항 정의서 (PRD)

## 프로젝트 개요

- **기능명**: 밤(夜) 설명서 — "은밀한 난상토론 유출 사건"
- **바이럴 공식**: [사주 기반 섹슈얼 진단 A] + [시종 3명이 유저를 두고 벌이는 쟁탈전 B] = "이런 건 처음이다"
- **바이럴 기능 간단 요약**: 사용자가 생년월일시를 입력하면 사주 기반으로 **은밀한 체질 평가표**(감도·지배력·중독성·민감도·지구력 5가지 스탯)가 생성된다. 이후 유저를 모시는 시종 3명(야수형/시인형/집사형)이 "오늘 밤 마마를 어떻게 모실 것인가"를 놓고 **난상토론**을 벌인다. 유저는 몰래 엿듣다가 개입하여 토론 흐름을 바꾸고, 각 시종이 유저의 체질에 맞는 밤 시중 방법을 제안한다. 최종적으로 가장 취향에 맞는 시종을 선택하면, 선택한 시종의 성격과 유저의 체질 조합으로 **밤 궁합 결산 카드**가 발급된다. "시종 3명이 자기 뽑아달라고 아우성인데 미쳤음ㅋㅋ 내 중독성 94래" — 이 한 줄이 트위터/여초 커뮤니티에서 캡처 공유되면서 "나도 해볼래" 유입을 폭발시킨다. 결산 후 선택한 시종이 X플레어 캐릭터로 연결되는 CTA로 유료 챗봇 전환한다.

## 바이럴 루프

### 바이럴을 일으키는 구조

1. **입력**: 생년월일시 (가입 불필요, 3초) → "밤의 장막이 내려오는 중..." 로딩 연출
2. **체질 평가표 생성**: 사주 기반 능력치 5종 + 도화살/홍염살 보유 여부 + 총 매혹력 → **1차 캡처 포인트**
3. **시종 3명 등장**: 유저 사주 상성에 따라 3명 배정. 각각 다른 성격·시중 스타일
4. **난상토론**: 시종 3명이 "마마를 어떻게 모실 것인가" 토론 → 유저 엿듣기 → 개입 → 토론 흐름 변화
5. **시종별 제안**: 각 시종이 유저 체질에 맞는 밤 시중 방법을 구체적으로 제안 → **2차 캡처 포인트 (명대사)**
6. **선택 + 결산 카드**: 취향에 맞는 시종 선택 → 밤 궁합 결산 카드 발급 → **바이럴 핵심 (3차 캡처 포인트)**
7. **전환**: 선택한 시종 = X플레어 캐릭터 → 유료 챗봇

### 유발 감정

| 감정 | 메커니즘 |
|------|---------|
| **호기심 + 수치쾌감 (Hook)** | "내 은밀한 체질이 뭐래?" — 성적 자아를 사주로 수치화하는 금기적 프레이밍 |
| **우월감 (Queen)** | 시종 3명이 나를 두고 쟁탈전 → 현실에서 저평가받는 타겟의 카타르시스 |
| **몰입 + 관음 (Voyeur)** | 시종들이 나를 놓고 토론하는 걸 엿듣는 쾌감 → "나에 대해 이렇게 말하네?" |
| **선택의 권력감 (Power)** | "오늘 밤 누가 나를 모실지 내가 정한다" → 현실 연애와 정반대 역전 |
| **자조적 유머 (Share)** | "AI 시종한테 선택받으려고 아등바등ㅋㅋ" / "중독성 94래 나 무섭다ㅋㅋ" |
| **비교 (Envy)** | "너 감도 몇이야?" / "누구 골랐어?" → 점수·선택 비교가 대화 소재 |

### 논란 설계

- **"AI가 내 섹슈얼 체질을 사주로 분석해준다"는 컨셉 자체가 화제**: 기존 사주 서비스에서 절대 안 하는 영역 → "이런 것도 됨?" 충격
- **시종 쟁탈전이 여성 판타지 직격**: 현실에서 선택받지 못하는 3040 여성 → 여기선 3명이 경쟁 → "어차피 판타지인데 뭐 어때" vs "이런 거 만들어도 되나" 논쟁
- **시종 대사 캡처가 밈화**: 맥락 없이도 자극적+웃긴 대사 → 트위터/여초에서 스크린샷 유통
- **스윗 스팟**: 사주+조선 세계관 포장 → 성인 콘텐츠지만 "운세 서비스"의 연장선 → 공격적이면서도 세계관 안에서 안전
- **체질 점수의 숫자가 밈**: "중독성 94" / "감도 38ㅋㅋ" → 구체적 숫자 = 공유 트리거

## 사용자 스토리

**As** (누가): 사회적 명성은 있지만 신체적 컴플렉스로 연애 시장에서 저평가받는 3040 여성 유저는

**Want to** (무엇을 원한다): 내 사주로 은밀한 체질 평가표를 받고, 시종 3명이 나를 두고 벌이는 쟁탈전을 엿들으며, 각 시종이 내 체질에 맞춰 제안하는 밤 시중 방법 중 가장 취향에 맞는 시종을 선택하고, "시종들이 날 두고 싸우는데 미쳤음ㅋㅋ" 결산 카드를 친구들과 공유하기를 원한다.

**So that** (어떤 목적/가치를 위해): "나는 선택받는 존재"라는 경험으로 현실에서 억눌린 성적 자존감을 회복하고, 쟁탈전에서 선택한 시종 캐릭터와 실제로 대화하고 싶은 동기로 사주 X플레어 챗봇 상담을 이어가기 위해.

## MVP 필요 기능

### Step 1. 체질 평가표 생성 — 사주 입력 1개면 끝

- 메인 화면: "밤(夜) 설명서 — 시종들의 은밀한 난상토론이 유출되었습니다" 문구 + "엿듣기" 버튼
- 입력 항목 4가지 (이것만으로 전체 흐름 구동):
  1. **성별**: 필수. 기본 "여성" 선택
  2. **생년월일**: 필수. 숫자 키패드 노출
  3. **태어난 시간**: 선택. "모름" 기본 선택
  4. **양력/음력**: 양력 기본 선택

#### 1-0. UTM 파라미터 자동입력 (사주GPT 제휴 유입)

사주GPT 광고 배너 클릭 시 아래 형식으로 유저 정보가 URL에 포함되어 유입된다:

```
/night-manual?utm_source=sajugpt&utm_medium=affiliate&utm_campaign=sdowha&birthday=199112252315&gender=male
```

**파라미터 파싱 및 자동입력 로직**:

```typescript
const params = new URLSearchParams(window.location.search);
const birthdayParam = params.get('birthday'); // "199112252315" (YYYYMMDDHHMI)
const genderParam = params.get('gender');     // "male" | "female"

if (birthdayParam && birthdayParam.length >= 8) {
  const year = birthdayParam.slice(0, 4);
  const month = birthdayParam.slice(4, 6);
  const day = birthdayParam.slice(6, 8);
  const hour = birthdayParam.slice(8, 10);
  const minute = birthdayParam.slice(10, 12);

  setBirthDate({ year, month, day });
  if (birthdayParam.length >= 10) {
    setBirthTime({ hour, minute });
  }
  setCalendarType('solar'); // 양력 기본값
}

if (genderParam) {
  setGender(genderParam === 'male' ? 'male' : 'female');
}
```

**핵심 규칙**:
- `birthday` 파라미터가 있으면 생년월일 + 시간 자동입력 → 유저는 확인만 하고 바로 시작
- `gender` 파라미터가 있으면 성별 자동입력
- 양력/음력은 **양력 기본값** 유지 (sajuGPT에서 양력으로 전달)
- 파라미터 없으면 기존 수동 입력 플로우 그대로

- 입력 직후 **"밤의 장막이 내려오는 중..."** 로딩 연출:
  - "사주 원국 펼치는 중..."
  - "도화살·홍염살 감지 중... 🔥"
  - "은밀한 체질 분석 중..."
  - "시종 3명 소집 중..."
  - → 기대감 빌드업 (2~3초)

#### 1-2a. 능력치 5가지 — API 필드 매핑

| 능력치 | 사주 API 근거 | 의미 | 바이럴 기능 |
|---|---|---|---|
| **감도** | `발달십성.인성` + `십성[]`에서 정인/편인 투출 + `12신살[]`에서 도화살 + `십이운성[]`에서 목욕 + `기타신살[]`에서 음욕살 | 자극에 대한 민감한 반응. 감각이 열리는 속도 | "감도 82래ㅋㅋ" 숫자 자랑 |
| **지배력** | `발달십성.관성` + `십성[]`에서 편관/정관/겁재 투출 + `본사주.양인살` + `십이운성[]`에서 제왕 + `본사주.괴강살` | 주도권을 쥐는 힘. 상대를 리드하는 기질 | "지배력 S인데 시종이 벌벌 떰ㅋㅋ" |
| **중독성** | `발달십성.식상` + `발달십성.재성` + `기타신살[]`에서 홍염살 + `십성[]`에서 상관/편재 투출 + `12신살[]`에서 도화살 + `기타신살[]`에서 음욕살 | 한 번 경험한 상대가 다시 찾게 만드는 기운 | "중독성 94래 나 무섭다ㅋㅋ" 핵심 밈 |
| **민감도** | `발달십성.식상` + `십성[]`에서 식신/정인 투출 + `12신살[]`에서 화개살 + `기타신살[]`에서 음욕살 | 미세한 변화를 감지하는 촉각. 분위기 읽기 | "민감도 높으면 시종이 당황함" |
| **지구력** | `발달십성.비겁` + `십성[]`에서 비견/겁재 투출 + `십이운성[]`에서 건록/관대/제왕 + `십성[]`에서 편관 투출 + `사주강약` | 밤이 깊어져도 흐트러지지 않는 체력 | "지구력 96 시종이 교대함ㅋㅋ" |

#### 1-3. 능력치 산정 로직 — 구현 상세

**판별 함수 정의:**

```typescript
// 십성 투출 판별: 십성 2D 배열 전체에서 해당 십성 존재 여부
function hasSipsung(sipsung: string[][], name: string): boolean {
  return sipsung.some(pair => pair.some(s => s === name));
}

// 십성 투출 횟수: 같은 십성이 여러 주에 나올 수 있음
function countSipsung(sipsung: string[][], name: string): number {
  return sipsung.flat().filter(s => s === name).length;
}

// 12신살 보유 판별: 4주 중 어디든 해당 살 존재
function has12Sinsal(sinsal12: string[][], name: string): boolean {
  return sinsal12.some(arr => arr.some(s => s.includes(name)));
}

// 기타신살 보유 판별: 4주 중 어디든 해당 살 존재
function hasGitaSinsal(gitaSinsal: string[][], name: string): boolean {
  return gitaSinsal.some(arr => arr.some(s => s.includes(name)));
}

// 십이운성 보유 판별: 4주 중 어디든 해당 운성
function has12Unseong(unseong: string[], name: string): boolean {
  return unseong.some(u => u === name);
}

// 본사주 살 보유 판별: 빈 문자열이 아니면 보유
function hasBonjuSal(bonju: Record<string, string>, key: string): boolean {
  return bonju[key] !== undefined && bonju[key] !== '';
}
```

**능력치 산정 공식:**

```
각 능력치 = Base + 투출 보너스 + 신살 보너스
  → Math.min(100, Math.max(10, 결과))

════════════════════════════════════════════════
감도 (Sensitivity) — 자극 반응 속도
════════════════════════════════════════════════
  Base:     발달십성.인성 × 1.2                  (0~60)
  Floor:    15 (인성 0이어도 최소 15)

  투출 보너스:
    정인 투출 (hasSipsung)         +12
    편인 투출                      +10
    정인 2회 이상 (countSipsung≥2) +5  (추가)

  신살·운성 보너스:
    도화살 (12신살)                +15
    음욕살 (기타신살)              +12
    목욕 (십이운성)                +10

  → 예상 범위: 15~95
```

### Step 2. 시종 3명 등장 — 난상토론 시작

체질 평가표 생성 직후, 시종 3명이 유저의 평가표를 들고 "오늘 밤 마마를 어떻게 모실 것인가" 토론을 벌인다.

#### 2-1. 시종 프로필

| 시종 | 유형 | 성격 | 시중 스타일 | 토론 화법 |
|---|---|---|---|---|
| **강해** | 🔥 야수형 | 직설적, 거침없음. 몸으로 증명하는 타입 | 본능적·압도적 | "말이 필요합니까? 제가 마마의 도화살을 제대로 받아내겠습니다" |
| **윤서** | 🎭 시인형 | 감성적, 말이 무기. 분위기를 만드는 타입 | 감각적·서사적 | "마마의 심화 체질은 급하게 다루면 안 됩니다. 천천히 불을 지펴야..." |
| **도겸** | 🫅 집사형 | 절제된 매너, 완벽한 준비. 복종하는 타입 | 헌신적·순종적 | "마마의 편안함이 최우선입니다. 제가 모든 것을 준비해놓겠습니다" |

#### 2-2. 토론 구조 — 유저 개입 포함

```
Phase 1: 엿듣기 (유저는 병풍 뒤에서 관찰)
────────────────────────────────────────
  시종 3명이 유저의 체질 평가표를 놓고 토론 시작
  → 유저는 스크롤하며 대화를 읽음
  → 시종들은 유저가 듣고 있는 줄 모름

Phase 2: 개입 (유저가 병풍을 넘어 등장)
────────────────────────────────────────
  토론이 과열될 때 유저 선택지 등장
  → "가만히 더 듣는다" / "끼어든다" / "헛기침으로 존재감을 알린다"
  → 선택에 따라 시종들의 반응과 이후 제안 톤이 달라짐

Phase 3: 시종별 최종 제안 (유저 체질 맞춤)
────────────────────────────────────────
  각 시종이 유저의 체질에 맞는 밤 시중 방법을 구체적으로 제안
  → 유저의 최고 능력치에 따라 제안 내용이 변동
  → 3개 제안 중 가장 취향에 맞는 것을 선택
```

#### 2-3. 토론 대사 — 캡처 포인트 설계

시종 대사는 **스크린샷 1장으로 맥락 없이도 자극적+웃긴** 수준이어야 한다.

**Phase 1: 엿듣기 — 시종들이 평가표를 보며 토론**

```
강해(야수형):
  "중독성 94? 이건... 제가 아니면 감당 못 합니다.
   윤서, 네 시 따위로 마마의 도화살을 어찌 하겠다는 거냐."

윤서(시인형):
  "무식한 놈. 마마의 민감도가 71이다. 세밀하게 다뤄야 해.
   네 식으로 덤벼들면 마마가 재미없어서 하품하실 거다."

도겸(집사형):
  "두 분 다 틀렸습니다. 마마의 감도 82를 보십시오.
   천천히, 정성껏, 한 겹씩 벗겨드려야 합니다.
   기다림이 최고의 시중입니다."

강해:
  "야, 지구력 96 봐. 마마는 기다림 따위 원하시지 않아.
   밤새 버틸 수 있는 건 나뿐이다."

윤서:
  "...솔직히 지구력 96은 나도 좀 무섭다."

도겸:
  "저는 마마께서 원하시면 무릎이 닳도록이라도..."
```

> 이 대화 중 "솔직히 지구력 96은 나도 좀 무섭다" 같은 한 줄이 **맥락 없이 캡처해도 웃긴 대사** → 트위터 스크린샷 유통

**Phase 2: 유저 개입 선택지**

| 선택 | 시종 반응 |
|---|---|
| **"가만히 더 듣는다"** | 토론이 더 과격해짐. 시종들이 서로 도발 → 더 자극적인 대사 등장 |
| **"헛기침으로 존재감 알린다"** | 시종들 경악 → "마, 마마?!" → 급히 자세 바꿈 → 코믹 전환 |
| **"직접 끼어든다"** | "뭐가 그리 시끄럽냐" → 시종들 즉시 무릎 → "명하십시오, 마마" |

> 선택에 따라 Phase 3의 제안 톤이 달라짐:
> - "더 듣는다" → 시종들이 경쟁심 극대화 → 제안이 더 과감
> - "헛기침" → 시종들이 당황 → 제안이 안절부절+귀여움 섞임
> - "직접 끼어든다" → 시종들이 순종 모드 → 제안이 복종적

#### 2-4. Phase 3: 시종별 최종 제안 — 유저 체질 기반 분기

유저의 **최고 능력치**에 따라 각 시종의 제안 내용이 달라진다.

**예시: 중독성 최고일 때**

```
강해(야수형):
  "마마의 중독성이 94라 함은... 한 번으로 끝나지 않는다는 뜻.
   제가 마마의 체력이 닳을 때까지 놓지 않겠습니다.
   아침에 일어나면 제 이름밖에 생각나지 않을 것입니다."

윤서(시인형):
  "중독이란 서서히 스미는 것입니다. 마마.
   오늘 밤 저는 마마의 손끝에서 시작하여 새벽녘에
   마마의 입술에 도착하겠습니다. 그 사이의 모든 곳에
   제 시를 새기겠습니다."

도겸(집사형):
  "마마의 중독성이 높을수록 상대가 지치기 쉽습니다.
   하지만 저는 지치지 않습니다. 마마께서 '그만'이라
   하실 때까지, 마마께서 '그만'이라 하셔도,
   한 번 더 여쭙겠습니다 — '정말이십니까?'"
```

> **"마마께서 '그만'이라 하셔도, 한 번 더 여쭙겠습니다"** 같은 대사가 캡처 밈 핵심

#### 3-2. 밤 궁합 등급 산정

유저 체질과 선택한 시종의 조합에 따라 **궁합 등급**이 결정된다.

| 유저 최고 능력치 | 야수형 궁합 | 시인형 궁합 | 집사형 궁합 |
|---|---|---|---|
| 감도 최고 | B (과하면 부담) | **S** (감각 극대화) | A (헌신적 케어) |
| 지배력 최고 | A (힘 대 힘 긴장) | B (리드 충돌) | **S** (완벽한 순종) |
| 중독성 최고 | **S** (밤새 중독 루프) | A (서사적 중독) | A (돌봄 중독) |
| 민감도 최고 | C (너무 거침) | **S** (섬세한 터치) | A (세심한 배려) |
| 지구력 최고 | **S** (체력 매치) | B (시인이 먼저 지침) | A (교대 서빙) |

> S궁합이 아닌 조합을 고르면 → "왜 이 시종을 골랐는지" 해석 텍스트 제공 + 리플레이 유도("S궁합 시종은 따로 있다")

```
    ↓

    ↓
```

**전환 심리 설계**:
1. 난상토론에서 시종에 대한 감정적 투자가 쌓임 (단순 테스트보다 깊은 몰입)
2. 탈락 시종의 아쉬운 반응 → "이 시종도 궁금하다" → 서브 CTA로 추가 전환
3. CTA가 토론 서사의 자연스러운 연장선 ("진짜로 밤을 보낸다")
4. 첫 3턴 무료 = Opt-out Trial 구조 (전환율 49.9% 벤치마크)
