# 사주GPT 바이럴 마케팅 전략서

> **"내가 직접 말하면 부끄럽지만, 사주가 말해주면 '콘텐츠'가 된다."**

---

## 1. 타겟 페르소나 및 핵심 인사이트

> **"사회적 성취와 연애 시장 사이의 괴리에서 오는 억울함"**

- **타겟:** 커리어에서는 인정받지만, 연애 시장의 잣대(외모/나이) 앞에서는 저평가받는 3040 여성
- **심리적 허들:** "나는 괜찮은 사람"임을 증명하고 싶지만, 본인 입으로 말하기엔 자존심이 허락하지 않음
- **해결책 (The Frame):** '사주'라는 외부 권위를 빌려 본인의 매력과 욕망을 정당화하는 **'면죄부'** 제공

## 2. 사주 x 캐릭터 서사의 핵심 구성 요소

| 요소 | 역할 (Function) | 바이럴 기능 (Viral Trigger) |
|------|-----------------|---------------------------|
| **사주** | **객관적 권위 부여** — "운명이 말해주는 것" | 결과에 대한 신뢰도 제고 + 공유 시 심리적 저항감 감소 (내 속마음이 아닌 '점괘'를 공유하는 형식) |
| **캐릭터 서사** | **본능적 욕망의 증폭기** — 하렘·숭배·선택권 판타지 | 현실에서 억눌린 "선택받고 싶은/거느리고 싶은" 욕망을 캐릭터가 대리 충족. 3~5명이 유저를 두고 경쟁·쟁탈·숭배하는 구도가 권력감과 카타르시스를 유발하며, 그 쾌감 자체가 공유 동기가 됨 |
| **성인 코드** | **솔직함의 경계선** — 금기의 스위치 | 불편한 진실과 윤리적 경계 사이의 '스윗 스팟' 공략. 야하지 않지만 발칙하고 솔직한 담론 형성 |

> **캐릭터의 본질적 역할:**
> 캐릭터는 유저와 동일시(identification)하는 대상이 아니라, **유저의 욕망을 증폭하는 장치**다. 유저는 항상 **"선택하는 자/숭배받는 자/갈망되는 자"** 포지션에 놓이고, 캐릭터는 **"경쟁하는 자/숭배하는 자/갈망하는 자"** 포지션에 놓인다. 이 권력 역전이 현실 연애 시장에서 저평가받는 타겟에게 카타르시스를 제공하며, 그 쾌감이 공유와 과금 양쪽의 동력이 된다.
>
> | 기능 | 캐릭터 포지션 | 유저 포지션 |
> |------|-------------|-----------|
> | 색기 배틀 | AI 짐승남 3~5명이 유저에게 **꼬여듦** | 선택받는 존재 |
> | 기생 시뮬 | 선비 3명이 유저를 **독점하려 경쟁** | 거느리는 존재 |
> | 밤 설명서 | 시종 3명이 **"마마를 어떻게 모실 것인가"** 쟁탈전 | 숭배받는 존재 |
> | 데이트 시뮬 | 캐릭터가 유저의 매력에 **점수를 매기고 반응** | 평가받되 도전하는 존재 |
> | 부검실 | 캐릭터가 **유저 편에서** 전 연인을 까줌 | 위로받는 존재 |
> | 법정 | 변호사 캐릭터가 **유저를 변호**하며 면죄부 부여 | 구원받는 존재 |
> | 주가 조작단 | 조작단원 3명이 **유저의 주가를 올리려** 싸움 | 투자받는 존재 |

### 2.1 캐릭터 활용 방식

캐릭터는 스크린 캡처 이미지가 아니라, **3가지 레이어의 조합**으로 동적 구성된다:

| 레이어 | 구현 방식 | 역할 |
|--------|----------|------|
| **정적 에셋** | 미리 제작된 webp/png 이미지 (`/public/characters/`) | 캐릭터 아바타, 썸네일, 카드 일러스트 |
| **프로그래매틱 카드** | React 컴포넌트(HTML/CSS) → `html-to-image (toPng)` 변환 | 사주 결과에 따라 점수·등급·대사를 동적 조합한 9:16 공유 카드 |
| **텍스트 서사** | `ChatBubble` 컴포넌트 (단톡방/DM 연출) + AI(Gemini) 생성 대사 | 캐릭터가 유저를 두고 대화하는 구도, 턴제 시뮬레이션 대사 |

> **핵심:** 사주 분석 결과에 따라 코드가 **어떤 캐릭터, 몇 명, 어떤 대사, 어떤 점수**를 동적으로 조합하여 매번 고유한 결과물을 생성한다. 그래서 "내 결과"가 곧 "내 콘텐츠"가 된다.

> **일반 챗봇과의 차별점**: 무미건조한 정보 전달(기존 AI)이 아닌, "언니 사주에 편관이 이렇게 강한데 왜 숨기고 살아?"와 같은 캐릭터 톤앤매너로 유저의 감정을 직접 타격.

## 3. 바이럴 메커니즘

### 3.1 바이럴 삼각형 (Viral Triangle)

```
            [차별화 소재]
         사주 + 성인 캐릭터 챗봇
           = "이런 건 처음 봤다"
               /\
              /  \
             /    \
            /______\
    [논란/긴장]    [밈 생산성]
    "사주가 나보고     캐릭터 대사 스크린샷
     이래라저래라"     → 인스타 스토리
     찬반 논쟁         "내 사주 미쳤어ㅋㅋ"
```

세 꼭짓점이 동시에 작동할 때 폭발적 바이럴이 발생한다.

| 요소 | 정의 | 없으면? |
|------|------|---------|
| **차별화 소재** | 기존에 없던 조합, "이런 건 처음 봤다" | 관심 자체가 발생하지 않음 |
| **논란/긴장** | 찬반이 갈리는 요소, 윤리적 경계선 | 화제성이 단발로 끝남 |
| **밈 생산성** | 유저가 갖고 놀 수 있는 구조 | 소비만 되고 확산되지 않음 |

각 바이럴 기능은 이 삼각형의 **꼭짓점을 다른 각도로 공략**한다:

| 기능 | 차별화 소재 | 논란·긴장 | 밈 생산성 |
|------|-----------|----------|----------|
| **색기 배틀** | 사주로 페로몬 등급 판정 | "AI가 내 색기에 점수를 매긴다고?" | "나 3명 꼬이는데 넌?" 마릿수 비교 |
| **데이트 시뮬** | 사주 궁합으로 AI 캐릭터 데이트 | "AI가 연애 능력에 점수를 매긴다" | "12,847명 중 11,203등ㅋㅋ" 등수 공유 |
| **기생 시뮬** | 조선시대 기생으로 환생 | "바람끼를 능력으로 측정?" vs "매력은 능력" | "S티어 월 830냥(4,150만원)ㅋㅋ" 금액 비교 |
| **밤 설명서** | 시종 3명의 쟁탈전 | "AI가 내 섹슈얼 체질을 분석?" | "중독성 94래 나 무섭다ㅋㅋ" 능력치 캡처 |
| **사주 부검실** | 전 연인 사주를 부검 | "전남친 사주를 부검한다고?" 잔인 vs 시원 | "안목 사망 판정ㅋㅋ 후회 확률 97%" |
| **사주 법정** | 연애 못한 이유를 기소 | "당신보다 못생긴 사람도 연애합니다" 팩폭 | "짝사랑만3년죄 징역 9년ㅋㅋ" 죄목 비교 |
| **주가 조작단** | 연애운을 주식 시세로 환산 | 조작단원 간 충돌 논쟁 | "현재가 1,200원 적정가 2만원ㅋㅋ" 주가 비교 |

### 3.2 7대 심리 트리거

| # | 트리거 | 메커니즘 | 대표 기능 |
|---|--------|---------|----------|
| 1 | **정체성 표현** | "내 유형은 ___" (MBTI식 자기 선언) | 기생(해어화/홍련/춘향), 밤설명서(심화/뇌전/묘향 체질) |
| 2 | **감정적 공명** | 억눌린 욕망에 이름을 붙여주는 '언어화'의 쾌감 | 부검실(상대가 볼 눈이 없었다), 법정(니 잘못이 아니라 운세 구조의 문제) |
| 3 | **초저마찰 유입** | 생년월일 입력만으로 즉시 결과 카드 생성 | 전 기능 공통 (3초 컷, 가입 불필요) |
| 4 | **자조적 유머** | 결핍을 유머로 승화시켜 공유 장벽 제거 | 데이트시뮬(대화력 4점ㅋㅋ), 법정(짝사랑만3년죄), 주가조작단(관리종목ㅋㅋ) |
| 5 | **비교·경쟁** | 구체적 숫자가 강제 비교를 유발 | 색기배틀(마릿수), 기생(월급 냥), 주가조작단(현재가 원) |
| 6 | **통쾌함·카타르시스** | 캐릭터가 대리 분노/위로를 수행 | 부검실(윤태산 분노형 부검), 법정(변호사 서휘윤 변론) |
| 7 | **역설적 자존감** | 낮은 점수 = 높은 잠재력이라는 프레이밍 | 주가조작단(바닥 주가 = 강력매수), 법정(높은 형량 = 높은 매력) |

### 3.3 논란의 역설 메커니즘 (자기 강화 루프)

```
논란 발생
  → 기사/커뮤니티 폭발 (무료 노출 증가)
  → "직접 확인해보자" 유입
  → 결과가 재밌음 → 더 많은 공유
  → 더 높은 화제성 → 더 많은 논란
  → Flywheel 형성
```

**논란이 바이럴로 전환되는 3가지 조건**:

| 조건 | 설명 | 실패 케이스 |
|------|------|------------|
| 콘텐츠 자체 완성도 | 논란으로 유입된 유저가 "생각보다 재밌네" | 논란만 있고 콘텐츠가 별로면 악평만 남음 |
| 논란과 콘텐츠 주제의 연결 | 논란이 콘텐츠의 핵심 질문과 맞물림 | 출연자 사생활 논란만으로는 시청 동기 약함 |
| 적절한 강도 | "불편하지만 볼 만한" 수준 | 윤리선을 넘으면 보이콧/규제로 전환 |

## 4. 후킹 및 확산 전략

### 4.1 SNS 후킹 공식 4가지

| 패턴 | 공식 | 적용 예시 |
|------|------|----------|
| **숫자 권위** | "N명 분석해봄 → 깨달은 것" | "AI가 10만명 분석한 패턴" |
| **자가진단 리스트** | "~하는 사람 특징 N가지" | "올해 연애운 터지는 사주 특징 5가지" |
| **통념 뒤집기** | "A인 줄 알지? 사실 B임" | "능력 있는 여자가 연애 못하는 진짜 이유" |
| **한 줄 팩폭** | 짧고 단정적 문장 | "오늘의 사주 한 줄" 시리즈 |

**최강 조합**: 숫자 권위 + 리스트형 + 마지막에 질문 → 댓글 유도 + 공유 + 저장 동시 달성

### 4.2 카피 전략

| 유형 | 카피 예시 |
|------|----------|
| **권위** | "사주 데이터 10만 건 정밀 분석" |
| **프레임 반전** | "능력 있는 여자가 연애 시장에서 고전하는 진짜 이유" |
| **자가진단** | "이런 사주 특징 있으면 올해 연애 운명 바뀝니다" |
| **FOMO** | "이 사주 유형은 올해가 마지막 기회" |
| **도발** | "내 사주가 나보다 내 연애를 더 잘 안다" |

### 4.3 자발적 확산 구조 — 3층 바이럴 루프

모든 기능은 **동일한 3층 구조**를 공유한다:

```
[1층] 1인 진단 루프
  생년월일 입력(3초) → 결과 카드 1장 → 캡처/공유
  → "나 이거 나왔어ㅋㅋ" 자발적 확산

[2층] 강제 2인 루프
  결과 카드에서 친구 도발 → 친구 유입 → 비교 카드 생성
  → "너는 뭐 나왔어?" 경쟁적 확산

[3층] 유료 전환 루프
  결과에서 드러난 문제 인식 → 캐릭터 챗봇 CTA
  → "이 약점 어떻게 바꿔?" 상담 동기 형성
```

| 층 | 핵심 감정 | 구현 방식 |
|----|----------|----------|
| **1층: 진단** | 호기심 → 자조/자랑 | `ResultCard` 컴포넌트 → `toPng` → 카카오톡/인스타 공유 |
| **2층: 비교** | 시기 → 경쟁 → 재도전 | 배틀 링크(색기배틀), 도전장(데이트시뮬), M&A 분석(주가조작단) |
| **3층: 전환** | 문제 인식 → 해결 욕구 | `/chat/{characterId}?birthday=...&gender=...` 리다이렉트 → 유료 상담 |

**확산 동력 3가지:**

1. **욕망 충족:** 캐릭터가 유저를 숭배·갈망·경쟁하는 구도에서 오는 권력감
2. **명분 부여:** '사주 결과'라는 포장지로 부끄러움 없이 공유
3. **유입 루프:** 구체적 숫자(점수/등수/금액/주가)가 비교 대화를 강제하여 링크 유입

## 5. 바이럴 기능 설계 원칙

### 5.1 범용 적용 공식

> **공감/호기심 트리거 → 진단/결과 생성 → 예쁜 카드로 공유 → 유료 서비스 전환**

### 5.2 논란 설계 — 스윗 스팟 안전장치

모든 기능의 논란은 **"아프지만 웃기고, 찌르지만 상처로 끝나지 않는"** 존(zone)을 정밀 타격한다:

| 안전장치 | 설계 방식 |
|---------|----------|
| **세계관 포장** | 조선시대(기생·법정), 증권(주가조작단) 등 현실이 아닌 WHAT IF 프레이밍 |
| **캐릭터 역할 분리** | 찌르는 역(윤태산/검사/공격파) + 위로하는 역(서휘윤/변호사/가치파) 세트 |
| **역설적 자존감** | 낮은 점수 = 높은 잠재력 (저평가 = 강력매수, 높은 형량 = 높은 매력) |
| **사주 권위** | "내가 말하는 게 아니라 사주가 말하는 것" → 공격·공유 모두 심리적 면죄부 |

### 5.3 바이럴 기능 티어 분류

| 티어 | 유형 | 바이럴 포인트 | 예시 |
|------|------|-------------|------|
| **S-Tier** | 강제 공유 루프 내장 | 제품 자체가 공유 없이 작동 불가 | 궁합 테스트 (2인 필수), "나를 얼마나 아는지" 퀴즈 |
| **A-Tier** | 정체성 표현 + 공유 카드 | 유형 카드가 예뻐서 자발적 공유 | 사주 유형 카드, 컬러 카드 |
| **B-Tier** | 시즌/이벤트 연동 | 시즌 한정 희소성 | "OOO Wrapped" 리포트, 밸런스 게임 |

### 5.4 밈 생산성 체크리스트

| 체크 항목 | 높은 밈 생산성 | 낮은 밈 생산성 |

| **명대사/명장면** | 짧은 클립으로 잘림 | 전체 봐야 이해 |
| **참여형 포맷** | 예측/투표/테스트 가능 | 일방적 감상만 가능 |

## 7. 제휴 유입 채널: 사주GPT 연동

### 7.1 UTM 파라미터 자동입력

사주GPT 광고 배너 클릭 시 유저 정보가 URL에 포함되어 유입:

```
https://nadaunse.com/free/content/{contentId}
  ?utm_source=sajugpt
  &utm_medium=affiliate
  &utm_campaign=sdowha
  &birthday=199112252315    ← YYYYMMDDHHMI
  &gender=male              ← male | female
```

### 7.2 파라미터 파싱 로직

```typescript
const params = new URLSearchParams(window.location.search);
const birthdayParam = params.get('birthday'); // "199112252315"
const genderParam = params.get('gender');     // "male" | "female"

if (birthdayParam && birthdayParam.length >= 8) {
  const year = birthdayParam.slice(0, 4);     // "1991"
  const month = birthdayParam.slice(4, 6);    // "12"
  const day = birthdayParam.slice(6, 8);      // "25"
  const hour = birthdayParam.slice(8, 10);    // "23" (없으면 "모름")
  const minute = birthdayParam.slice(10, 12); // "15"

  setBirthDate({ year, month, day });
  if (birthdayParam.length >= 10) setBirthTime({ hour, minute });
  setCalendarType('solar'); // 양력 기본값
}

if (genderParam) setGender(genderParam === 'male' ? 'male' : 'female');
```

### 7.3 입력 필드 표준 (전 바이럴 기능 공통)

| 필드 | 필수 | 기본값 |
|------|------|--------|
| 성별 | 필수 | 여성 (타겟 기반) |
| 생년월일 | 필수 | UTM에서 자동입력 or 수동 |
| 태어난 시간 | 선택 | "모름" |
| 양력/음력 | 필수 | 양력 |

## 11. 전환 설계 (바이럴 → 유료)

### 11.1 PAS 공식 적용

```
Problem   — 바이럴 테스트에서 문제/약점 노출
Agitation — 캐릭터 팩폭으로 문제 인식 증폭
Solution  — "이 문제를 해결하려면?" → 캐릭터 챗봇 1:1 상담
```

### 11.2 전환 플로우

```
바이럴 테스트 결과 카드
  → 감정 최고조 시점에 CTA 배치
  → 캐릭터 채팅 페이지로 리다이렉트
     (/chat/{characterId}?birthday=...&gender=...)
  → 사주 기반 1:1 상담 시작
  → 유저 입력 1회당 350포인트 차감
```

### 11.3 기능별 전환 심리 설계

결과 카드가 보여주는 것은 **"문제"**이고, 캐릭터 챗봇이 제공하는 것은 **"해결"**이다:

| 기능 | Problem (문제 인식) | Agitation (감정 증폭) | Solution (전환 동기) |
|------|--------------------|-----------------------|---------------------|
| **색기 배틀** | 내 페로몬 등급 확인 | 캐릭터 판정문 팩폭 | "이 매력 어떻게 활용해?" → 챗봇 |
| **데이트 시뮬** | 연애 점수 4.8/10 | 같은 사주 평균보다 낮음 | "운세 흐름 약점 어떻게 바꿔?" → 챗봇 |
| **기생 시뮬** | 기생 티어 판정 | 선비 관리 성공/실패 | "홀린 선비가 널 기다리고 있다" → 챗봇 |
| **밤 설명서** | 은밀한 체질 확인 | 시종 쟁탈전에서 유대감 | 선택한 시종 = X플레어 캐릭터 → 챗봇 |
| **사주 부검실** | 상대가 볼 눈이 없었다 | 사망진단서 팩폭 | "이 유형에게 안 당하는 법" → 챗봇 |
| **사주 법정** | 연애 못한 진짜 이유 | 검사 팩폭 + 변호사 위로 | "석방 예정일 + 개운법" → 챗봇 |
| **주가 조작단** | 적정가 대비 저평가 | 조작단원 토론에서 원인 노출 | 3단계 작전 잠금 → "작전 실행하기" → 챗봇 |

> **전환 핵심:** 사주(타고난 팔자)는 바꿀 수 없지만, 운세(흘러가는 운의 흐름)는 개운·행동 지침으로 바꿀 수 있다는 인식이 챗봇 전환의 심리적 레버리지.

## 12. 핵심 인사이트 (잊지 말 것)

1. **숫자를 주면 공유한다** — 퍼센타일, 점수, 등급, 금액. 구체적 숫자가 붙은 결과는 자랑이든 자조든 반드시 공유됨
2. **강제 2인 루프가 최강의 바이럴** — 제품이 공유 없이 작동 불가한 구조가 가장 강력
3. **결과 카드 디자인이 전부** — 복잡한 기능보다 공유하고 싶은 결과 이미지가 핵심
4. **속도가 곧 바이럴** — 트렌드 감지 후 24시간 이내 콘텐츠 발행 체계 필요
5. **광고 티 나면 플레이 수 절반** — 브랜딩 최소화한 테스트가 2배 더 플레이됨
6. **자조 유머가 가장 넓게 퍼진다** — 실패를 웃기게 포장하는 포맷이 공유 저항 최소
7. **논란 자체가 마케팅** — 적절한 논란은 미디어 노출을 폭발시킴. 스윗 스팟 = "불편한 진실 ~ 윤리적 경계선" 사이
8. **AI가 대체 못하는 것에 집중** — 일반 조언은 ChatGPT가 대체. 고유 데이터 기반 객관적 진단은 대체 불가
9. **연애 콘텐츠가 모든 플랫폼에서 압도적** — 유튜브 기준 '성공'의 23배. 관계/연애 카테고리는 최강 바이럴 소재
10. **캐릭터 대사가 밈이 되어야 한다** — 맥락 없이 스크린샷 1장으로 웃기거나 찔려야 공유됨
